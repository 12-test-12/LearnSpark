这个「开源知识库系统搭建方案」思路清晰、架构分层合理，特别是多格式解析、智能搜索、权限与版本控制部分，完全可以作为学习平台的“知识中台”来用。把它和你前面的需求结合，整个产品就从“计划勉励工具”升级成了“AI驱动的学习计划+个人知识库+勉励系统”的融合体。下面我给出一个融合后的整体产品搭建稿，直接可落地。

一、产品定位与目标
产品名称暂定：LearnSpark（灵犀学习）
一句话定义：AI 辅助制定学习路线、审核每日成果，并帮你把学习资料和总结沉淀为可检索的个人知识库，配合提醒与勉励，形成“计划→执行→沉淀→正向反馈”的闭环。

核心价值：

主动规划：AI 结合网络资源与个人资料生成定制化学习路线。

执行监督：每日任务提交后 AI 审核，给出鼓励性评语。

知识沉淀：所有上传资料、提交总结自动解析入库，支持语义搜索和双向链接。

提醒与激励：邮件/通知提醒，打卡日历、成就徽章等情感化设计。

关键点：

前端在 AI 设置页配置自己的 Key，并选择密钥管理模式。

三、核心功能模块详细设计
1. 个人知识库（文档解析 + 智能搜索）
   功能：

用户上传的学习资料（PDF/Word/Markdown/图片等）自动解析入库，提取全文、标题、标签。

用户每次提交的任务总结（文字+截图描述）也自动存入知识库，形成“学习日志”。

支持Obsidian风格的双向链接：解析 Markdown 中的 [[笔记名]]，自动关联。

全文搜索（Elasticsearch） + 语义搜索（向量检索，FAISS/Chroma），可检索历史笔记、上传资料。

基于文档片段的相似推荐：“你正在学 React Hooks，知识库中有相关笔记3篇，推荐回顾”。

技术实现：

解析引擎：Apache Tika（通用）+ Flexmark（Markdown增强）。

向量化：调用 DeepSeek Embedding 或本地模型，将文档片段转为向量存入 PGVector/FAISS。

存储：MinIO 存原始文件，Elasticsearch 存文本索引，PostgreSQL 存元数据和向量。

与学习计划的结合：

生成路线时，AI 不仅搜索网络，还会搜索用户个人知识库中的相关笔记，作为“已有基础”，避免重复学习。

审核提交物时，可将用户总结与知识库中以往笔记对比，给出“你已掌握该概念，本次总结更深入”等反馈。

2. 学习路线生成（网络搜索 + 知识库 + 上传资料）
   流程：

用户输入目标、每日可用时长，可选上传资料。

后端生成 3~5 个搜索 query，并行调用 Bing Search API（需用户提供 Key）和个人知识库搜索（Elasticsearch/向量）。

上传资料经 Tika 解析成文本。

将所有上下文（目标、时长、搜索结果摘要、知识库相关笔记片段、上传资料内容）拼装成 Prompt，调用 DeepSeek（用户 Key）。

DeepSeek 返回结构化 JSON 路线（阶段→每日任务），后端解析后创建计划与任务，前端展示可编辑时间轴。

示例 Prompt 片段：

text
你是一位学习路径规划专家。请根据以下信息制定学习路线并拆解为每日任务。
目标：{{goal}}，每天学习{{hours}}小时。
网络搜索结果：{{web_results}}
用户个人知识库相关笔记摘要：{{kb_notes}}
用户上传资料内容：{{upload_content}}
要求：输出JSON，包含phases数组，每个phase有name、objective、tasks(day,description,verification_criteria)。
3. AI 每日审核与勉励
   流程：

用户提交任务总结（文字必填，可附截图链接）。

服务端组装 Prompt：任务描述 + 验证标准 + 用户总结，调用 DeepSeek 返回 { passed, score, feedback }。

审核通过则任务完成，积累积分；不通过则展示改进建议，可重新提交。

反馈内容自动存入知识库（关联任务、日期），并触发一句随机鼓励语。

情感化设计：

审核通过时弹出“太棒了！AI 给你 9 分”的动画。

连续通过 N 天解锁特殊徽章。

4. 提醒与通知
   邮件提醒：用户设置每日提醒时间，Spring Boot 定时任务扫描未完成任务用户，使用 Thymeleaf 模板发送邮件，内容包含今日任务列表、昨日 AI 鼓励语。

浏览器通知：Service Worker 推送（需用户授权）。

未来扩展：短信、飞书/企业微信通知。

5. 权限与协作（轻量版，可后期扩展）
   个人知识库默认为私有空间。

可创建“学习小组”，将指定计划或知识库空间共享给组内成员，支持只读/评论权限（RBAC 模型简化）。

小组内可互相查看进度（匿名显示积分），营造共同勉励氛围。

   获取用户的所有项目列表

AI 生成路线（核心）
POST /projects/{id}/generate-plan
请求（multipart/form-data 或 JSON）：

uploadFiles (可选，文件)

useWebSearch (boolean)

searchApiKey (若用户配置为空且本次想使用，可临时传入)
处理流程：

解析上传文件 → 提取文本
如果 useWebSearch=true，生成搜索 queries，调用搜索 API
查询用户知识库相关条目（向量相似度）
组装 Prompt 调用 DeepSeek，获取 JSON
解析 JSON 写入 phases 和 tasks 表
返回生成的完整路线（phases 和 tasks）
PUT /projects/{id}/plan
手动调整路线（排序、增删改任务）

4. 任务与审核
   GET /projects/{id}/tasks?date=2026-07-12

POST /tasks/{id}/submit

获取任务描述、验证标准

将提交内容自动写入知识库（source_type='submission'）
返回审核结果：{ passed, score, feedback }
GET /tasks/{id}/submissions
查看历史提交记录

   │   │   └─ ProjectDetail  // 项目详情：路线时间轴、任务看板
   │   │       ├─ TaskSubmit // 任务提交审核页
   │   │       └─ GeneratePlan // AI生成路线向导
   │   ├─ KnowledgeBase     // 知识库浏览、搜索、笔记详情
   │   ├─ Settings          // 个人设置、AI配置、提醒设置

2. 关键页面设计要点
   (1) AI 配置页（Settings/AIConfig）
   选项卡：DeepSeek 配置 / 搜索 API 配置

DeepSeek 部分：

API Key 输入框（带显示/隐藏切换）

Base URL（默认值可修改）

模型选择（下拉框，可自定义）

本地模式开关：说明提示“开启后密钥仅存于浏览器，部分功能可能受限”

搜索部分：

Bing Search API Key 输入框

测试按钮：输入测试查询，验证 Key 是否有效

保存按钮，成功后提示“配置已加密存储”或“本地模式已启用”

(2) 仪表盘（Dashboard）
顶部卡片：「今日待完成 3 项」「连续打卡 15 天」「总积分 1280」

中间：学习日历热力图（类似 GitHub 贡献图）

右侧：AI 每日一句（调用 DeepSeek 生成激励语）

点击任务可直接跳转提交

(3) 项目详情与路线（ProjectDetail）
上部：项目名称、目标、进度条

左侧：路线时间轴（使用 Naive UI 的 Timeline 组件），每个阶段展开显示每日任务列表

任务状态用不同颜色标记：待完成（灰色）、已提交（蓝色）、通过（绿色）、未通过（红色）

悬浮任务卡片可快速「提交审核」

右上角按钮「AI 优化路线」「手动添加任务」

(4) AI 生成路线向导（GeneratePlan）
步骤1：输入学习目标，每日可用小时数

步骤2：上传参考资料（拖拽上传，支持批量）

步骤3：选项开关「使用网络搜索增强路线」，提示需配置搜索 API

点击「生成学习路线」，显示进度动画（可展示 AI 思考过程）

生成完毕：展示可编辑的路线时间轴，用户可以拖拽任务排序，修改描述，确认后一次性保存

(5) 任务提交与审核（TaskSubmit）
显示任务描述、验证标准

输入框：学习总结（Markdown 编辑器，推荐 @kangc/v-md-editor）

附件：可粘贴图片链接或上传截图到 MinIO（返回链接填入）

提交后即时显示审核结果卡片：

通过：绿色+撒花动画+鼓励语“你太棒了！AI 给了 9 分”

未通过：黄色+改进建议，可「重新提交」

(6) 知识库（KnowledgeBase）
搜索栏支持全文搜索（关键词）和语义搜索（自然语言），切换搜索模式

搜索结果列表：标题、摘要、标签、来源（上传/提交）、创建时间

点击笔记进入详情页：

渲染 Markdown 内容

双向链接以标签形式显示，点击跳转

相关推荐（基于向量相似度）

侧边栏：标签云、最近编辑

初始化 Spring Boot，数据库迁移（Flyway），实现 JWT 认证。

实现核心闭环
