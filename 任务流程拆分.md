# LearnSpark 任务流程拆分

> 基于《任务书.md》拆解的可执行任务清单，按阶段 → 模块 → 任务三级组织。
> 每个任务包含：编号、描述、所属层、前置依赖、验收标准、预估工时。
> 编号规则：`阶段.模块.序号`（如 `1.2.3` = 阶段一 / 模块2 / 任务3）

---

## 一、项目概览

| 项 | 内容 |
|---|---|
| 产品名称 | LearnSpark（灵犀学习） |
| 产品定位 | AI 辅助学习路线 + 个人知识库 + 勉励系统 |
| 前端 | Vue3 + Vite + Pinia + Naive UI（PWA） |
| 后端 | Spring Boot 3.x（初期单体模块化，后期可拆微服务） |
| 数据层 | MySQL 8.0 / Elasticsearch 8.x / MinIO |
| 文档解析 | Apache Tika 2.x + Flexmark |
| AI | DeepSeek（用户提供 Key，AES 加密存储） |
| 网络搜索 | Bing Web Search API v7 |
| 部署 | Docker Compose → K8s |

### 任务总览

| 阶段 | 周期 | 核心目标 | 任务数 |
|---|---|---|---|
| 阶段一 | 4~6 周 | 核心闭环 MVP（认证 + 计划 + 审核 + 最小知识库 + 提醒 + 部署） | 30 |
| 阶段二 | 2~4 周 | 智能生成路线（Bing 搜索 + ES 索引 + AI 路线生成 + 文件解析扩展） | 12 |
| 阶段三 | 2 周 | 体验打磨（双向链接 + 成就系统 + 主题动画 + 数据仪表盘） | 10 |
| 阶段四 | 按需 | 协作与生态（小组空间 + 移动端 + RAG 问答） | 9 |

## 二、阶段一：核心闭环 MVP（4~6 周）

### 模块 1.1 项目骨架搭建

#### 任务 1.1.1 后端项目初始化
- **描述**：创建 Spring Boot 3.x 单体项目，按模块化划分包结构（auth / plan / ai / kb / notification / gamification），引入 Flyway、JPA、MySQL 驱动、Lombok、validation、Spring Security。
- **层**：后端
- **前置**：无
- **验收**：项目可启动，健康检查接口 `/actuator/health` 返回 UP，Flyway 能连接空库。
- **工时**：0.5 天

#### 任务 1.1.2 前端项目初始化
- **描述**：Vite 创建 Vue3 + TS 项目，引入 Pinia、Vue Router、Naive UI、Axios、v-md-editor、PWA 插件（vite-plugin-pwa）。搭好 Layout（顶栏 + 侧边栏 + 内容区）、路由表、请求拦截器、Token 持久化。
- **层**：前端
- **前置**：无
- **验收**：`npm run dev` 启动，能看到带导航的空布局页；PWA manifest 可识别。
- **工时**：1 天

#### 任务 1.1.3 数据库初始化与基础迁移
- **描述**：本地装 MySQL 8.0；编写 Flyway 初始迁移脚本 `V1__init.sql`，建好 `users`、`user_ai_config`、`projects`、`phases`、`tasks`、`submissions`、`knowledge_entries`、`knowledge_links`、`reminder_settings`、`user_scores`、`badges`、`user_badges`、`groups`、`group_members` 共 15 张表（字段按任务书定义）。
- **层**：数据库
- **前置**：1.1.1
- **验收**：`flyway migrate` 成功，所有表结构与外键约束正确。
- **工时**：1 天

#### 任务 1.1.4 基础设施 Docker 化
- **描述**：编写 `docker-compose.yml`，包含 mysql、minio、elasticsearch 服务；后端和前端也提供 Dockerfile。
- **层**：运维
- **前置**：1.1.1、1.1.2
- **验收**：`docker compose up` 一键拉起所有依赖服务，端口可访问。
- **工时**：0.5 天

### 模块 1.2 用户认证

#### 任务 1.2.1 用户注册登录接口
- **描述**：实现 `POST /api/v1/auth/register`、`POST /api/v1/auth/login`。密码用 BCrypt 哈希；登录成功签发 JWT。
- **层**：后端
- **前置**：1.1.1、1.1.3
- **验收**：注册 → 登录 → 拿到 token；重复邮箱注册返回 409；密码错误返回 401。
- **工时**：1 天

#### 任务 1.2.2 JWT 鉴权中间件
- **描述**：Spring Security + JWT Filter，配置白名单（auth 接口），其他接口需携带 `Authorization: Bearer <token>`；提供 `@CurrentUser` 注解获取当前用户。
- **层**：后端
- **前置**：1.2.1
- **验收**：无 token 访问受保护接口返回 401；带正确 token 能拿到当前用户信息。
- **工时**：1 天

#### 任务 1.2.3 前端登录注册页面
- **描述**：登录页、注册页（Naive UI Form + 校验）；Pinia 存 token 与用户信息；axios 拦截器自动携带 token，401 时跳登录。
- **层**：前端
- **前置**：1.1.2、1.2.1
- **验收**：注册→登录→进入 Dashboard；刷新页面登录态保持；登出后回登录页。
- **工时**：1 天

### 模块 1.5 AI 每日审核

#### 任务 1.5.1 审核接口（Mock 版）
- **描述**：`POST /api/v1/tasks/{id}/submit`，请求 `{ content, attachmentUrls }`；先实现 Mock 逻辑：固定返回 `{ passed: true, score: 9, feedback: "（Mock）" }`，写 `submissions` 表，更新任务状态为 passed/submitted，把内容写入 `knowledge_entries`（source_type='submission'）。
- **层**：后端
- **前置**：1.4.2
- **验收**：提交后任务状态变更；submission 记录入库；知识库出现对应条目；返回结构正确。
- **工时**：1 天

#### 任务 1.5.2 接入 DeepSeek 真实审核
- **描述**：用用户 AI 配置调用 DeepSeek；Prompt = 任务描述 + 验证标准 + 用户总结；要求模型返回 `{ passed, score(1-10), feedback }`；处理超时、限流、Key 无效等异常并降级。
- **层**：后端
- **前置**：1.3.2、1.5.1
- **验收**：真实提交能拿到 AI 评分与评语；Key 无效时返回明确错误；评分合理。
- **工时**：1 天

#### 任务 1.5.3 前端任务提交与审核结果页
- **描述**：TaskSubmit 页：展示任务描述与验证标准；v-md-editor 写总结；附件粘贴链接或上传截图；提交后展示 AIFeedback 卡片（通过→绿色+撒花动画+鼓励语；未通过→黄色+改进建议+重新提交按钮）。
- **层**：前端
- **前置**：1.5.1
- **验收**：提交流程顺滑；审核结果动画到位；未通过可重提；历史提交可查。
- **工时**：1.5 天

## 三、阶段二：智能生成路线（2~4 周）

### 模块 2.1 网络搜索集成

#### 任务 2.1.1 Bing Search API 封装
- **描述**：封装 Bing Web Search v7 客户端，支持多 query 并发（CompletableFuture）；返回结果摘要列表 `{ title, url, snippet }`；处理限流与 Key 异常。
- **层**：后端
- **前置**：1.3.2
- **验收**：传入多个 query 能并发返回结果；Key 无效时降级提示。
- **工时**：1 天

#### 任务 2.1.2 搜索 Query 生成
- **描述**：用 DeepSeek 根据用户目标生成 3~5 个搜索 query（JSON 数组）；Prompt 模板可配置。
- **层**：后端
- **前置**：2.1.1
- **验收**：输入目标能产出合理 query；可被 Bing 客户端消费。
- **工时**：0.5 天

### 模块 2.2 知识库搜索增强

#### 任务 2.2.1 Elasticsearch 接入与索引
- **描述**：建 ES 索引 `knowledge_entries`；上传/提交时同步写入 ES；提供关键词搜索实现（替代/补充 MySQL 全文检索）。
- **层**：后端
- **前置**：1.6.2、1.1.4
- **验收**：上传后 ES 中可搜到；搜索召回率与 MySQL 全文检索一致或更优。
- **工时**：1.5 天

#### 任务 2.2.2 向量化与语义搜索
- **描述**：调用 DeepSeek Embedding（或本地模型）将文档片段向量化，存入 JSON 向量字段（建议接入外部向量库 Chroma/Milvus）；`GET /api/v1/knowledge?q=&type=semantic` 用向量相似度召回；BM25 + 向量用 RRF 融合排序。
- **层**：后端
- **前置**：2.2.1、1.3.2
- **验收**：语义搜索能召回关键词不同但含义相近的条目；混合排序结果合理。
- **工时**：2 天

### 模块 2.3 AI 路线生成

#### 任务 2.3.1 路线生成核心接口
- **描述**：`POST /api/v1/projects/{id}/generate-plan`（multipart，支持上传文件 + `useWebSearch` 开关）；流程：解析上传文件 → 按需调搜索 → 查知识库相关笔记 → 拼 Prompt 调 DeepSeek → 解析 JSON `{ phases: [{ name, objective, tasks: [{ day, description, verification_criteria }] }] }` → 写入 phases/tasks 表 → 返回完整路线。
- **层**：后端
- **前置**：2.1.2、2.2.2、1.6.2
- **验收**：输入目标+时长能生成结构化路线；任务有验证标准；DB 中数据完整。
- **工时**：2 天

#### 任务 2.3.2 路线手动调整接口
- **描述**：`PUT /api/v1/projects/{id}/plan`，支持批量排序、增删改阶段与任务；事务保证一致性。
- **层**：后端
- **前置**：2.3.1
- **验收**：调整后 DB 与返回一致；排序正确。
- **工时**：0.5 天

#### 任务 2.3.3 前端 AI 生成路线向导
- **描述**：GeneratePlan 向导：Step1 目标+时长 → Step2 拖拽上传资料 → Step3 网络搜索开关 → 生成中显示 AI 思考动画 → 生成后可编辑时间轴（拖拽排序、改描述）→ 确认保存。
- **层**：前端
- **前置**：2.3.1、2.3.2
- **验收**：完整走完向导能生成并保存路线；可编辑；失败有提示与重试。
- **工时**：2 天

## 八、风险与对策（沿用任务书）

| 风险 | 对策 |
|---|---|
| 用户 Key 安全 | 前端模式不落服务端；后端模式 AES 加密 + 日志脱敏 + 审计 |
| AI 路线质量波动 | 「重新生成」按钮 + 手动调整 + 相同目标缓存 |
| 大文件解析性能 | 异步队列 + 限大小 + 超时控制 |
| 搜索精度 | BM25 + 向量 RRF 融合，后续可微调 embedding |
| 邮件进垃圾箱 | 对接 Resend 等专业服务 + 重试 |
