# LearnSpark 数据库设计方案

> 本文件是 LearnSpark 项目**唯一的数据库基准**。后续所有模块开发（认证 / 计划 / 审核 / 知识库 / 提醒 / 成就 / 协作）都以此为准，任何表结构变更必须先修改本文件并在 Flyway 中追加迁移脚本。
>
> - DBMS：PostgreSQL 15
> - 必装扩展：`pgcrypto`（UUID 生成）、`pgvector`（向量检索）、`pg_trgm`（模糊搜索）、`unaccent`（全文检索去重音）
> - 字符集：UTF8
> - 时区：所有时间字段使用 `TIMESTAMPTZ`，应用层统一 Asia/Shanghai 展示

---

## 一、设计原则与约定

| 项 | 约定 |
|---|---|
| 主键 | 全部使用 `UUID`，默认 `gen_random_uuid()`，避免自增 ID 暴露业务量 |
| 命名 | 表名小写蛇形复数（`users`）；外键字段 `<表名单数>_id`（`user_id`）；布尔字段 `is_xxx` / `enabled` |
| 时间 | `created_at` / `updated_at` 一律 `TIMESTAMPTZ DEFAULT NOW()`；`updated_at` 由触发器自动维护 |
| 软删除 | 核心业务表（projects / tasks / knowledge_entries / submissions）加 `deleted_at`，避免误删丢失 |
| 枚举 | 用 `VARCHAR + CHECK` 而非 PG enum，便于扩展（避免 enum 修改锁表） |
| 向量 | 统一 `vector(1536)`，若换 embedding 模型另起一列，旧列保留 |
| 数组 | 标签等少量多值字段用 `TEXT[]` + GIN 索引；多对多关系仍用关联表 |
| 索引 | 外键自动建索引；高频查询字段、全文检索、向量检索显式建索引 |

### updated_at 自动维护触发器

```sql
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

每张含 `updated_at` 的表建好后挂触发器：

```sql
CREATE TRIGGER trg_<table>_updated
  BEFORE UPDATE ON <table>
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

## 二、ER 总览

```
users ─┬── user_ai_config (1:1)
       ├── projects (1:N) ─┬── phases (1:N) ── tasks (1:N) ── submissions (1:N)
       │                   └── user_scores (1:N)
       ├── knowledge_entries (1:N) ── knowledge_links (N:N 自反)
       ├── reminder_settings (1:1)
       ├── user_badges (N:1) ── badges
       └── group_members (N:1) ── groups ── group_shares
```

核心数据流：**用户创建项目 → AI 生成阶段/任务 → 用户提交总结 → AI 审核写 submission → 同步落 knowledge_entries → 更新 user_scores → 触发徽章发放**。

### 4.1 用户与权限模块

#### 4.1.1 `users` —— 用户表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID PK | 用户 ID |
| email | VARCHAR(255) UNIQUE | 登录邮箱 |
| password_hash | VARCHAR(255) | BCrypt 哈希 |
| nickname | VARCHAR(100) | 昵称 |
| avatar_url | TEXT | 头像地址（MinIO） |
| timezone | VARCHAR(50) | 时区，默认 Asia/Shanghai |
| status | VARCHAR(20) | active / disabled |
| last_login_at | TIMESTAMPTZ | 最近登录时间 |
| created_at / updated_at | TIMESTAMPTZ | 审计字段 |

```sql
CREATE TABLE users (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email           VARCHAR(255) UNIQUE NOT NULL,
  password_hash   VARCHAR(255) NOT NULL,
  nickname        VARCHAR(100),
  avatar_url      TEXT,
  timezone        VARCHAR(50) DEFAULT 'Asia/Shanghai',
  status          VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active','disabled')),
  last_login_at   TIMESTAMPTZ,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_users_email_lower ON users (LOWER(email));
CREATE TRIGGER trg_users_updated BEFORE UPDATE ON users
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

**业务操作**：
- 写：1.2.1 注册/登录（insert / 更新 last_login_at）
- 读：1.2.2 JWT 解析、几乎所有接口的 `@CurrentUser`
- 后续：4.1 小组功能扩展 group_members 关联

### 4.4 知识库模块

#### 4.4.1 `knowledge_entries` —— 知识条目

| 字段 | 类型 | 说明 |
|---|---|---|
| source_type | VARCHAR(50) | upload / submission / manual |
| source_id | UUID | 上传文件 ID 或 submission ID |
| vector_embedding | vector(1536) | 语义检索向量 |

```sql
CREATE TABLE knowledge_entries (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id           UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  project_id        UUID REFERENCES projects(id) ON DELETE SET NULL,
  title             VARCHAR(500) NOT NULL,
  content           TEXT,                                    -- 解析后纯文本
  content_md        TEXT,                                    -- 原始 Markdown
  summary           TEXT,                                    -- AI 生成摘要（可选）
  source_type       VARCHAR(50) CHECK (source_type IN ('upload','submission','manual')),
  source_id         UUID,
  file_path         TEXT,                                    -- MinIO 路径
  mime_type         VARCHAR(100),
  tags              TEXT[] DEFAULT '{}',
  word_count        INT DEFAULT 0,
  parse_status      VARCHAR(20) DEFAULT 'done'
                    CHECK (parse_status IN ('pending','parsing','done','failed')),
  vector_embedding  vector(1536),
  search_tsv        tsvector,                                -- 全文检索列（触发器维护）
  deleted_at        TIMESTAMPTZ,
  created_at        TIMESTAMPTZ DEFAULT NOW(),
  updated_at        TIMESTAMPTZ DEFAULT NOW()
);

-- 全文检索：标题权重 A，正文权重 B，标签权重 C
CREATE INDEX idx_ke_search_tsv ON knowledge_entries
  USING GIN (search_tsv) WHERE deleted_at IS NULL;
-- 标签数组索引
CREATE INDEX idx_ke_tags ON knowledge_entries USING GIN (tags);
-- 向量索引（HNSW，pgvector 0.5+）
CREATE INDEX idx_ke_vector ON knowledge_entries
  USING hnsw (vector_embedding vector_cosine_ops) WHERE deleted_at IS NULL;
-- 用户维度浏览
CREATE INDEX idx_ke_user_created ON knowledge_entries(user_id, created_at DESC) WHERE deleted_at IS NULL;
-- 模糊搜索标题
CREATE INDEX idx_ke_title_trgm ON knowledge_entries USING GIN (title gin_trgm_ops);

-- search_tsv 维护函数
CREATE OR REPLACE FUNCTION ke_tsv_update() RETURNS trigger AS $$
BEGIN
  NEW.search_tsv :=
    setweight(to_tsvector('simple', unaccent(COALESCE(NEW.title,''))), 'A') ||
    setweight(to_tsvector('simple', unaccent(COALESCE(NEW.content,''))), 'B') ||
    setweight(to_tsvector('simple', unaccent(COALESCE(array_to_string(NEW.tags,' '),''))), 'C');
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_ke_tsv BEFORE INSERT OR UPDATE OF title,content,tags
  ON knowledge_entries FOR EACH ROW EXECUTE FUNCTION ke_tsv_update();

CREATE TRIGGER trg_ke_updated BEFORE UPDATE ON knowledge_entries
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

**业务操作**：
- 写：1.6.2 上传解析、1.5.1 提交时同步落库、2.2.2 向量化更新 vector_embedding、2.4.2 异步更新 parse_status
- 读：1.6.3 全文搜索、2.2.1 ES 同步源、2.2.2 语义搜索、2.3.1 路线生成时检索相关笔记、4.3 RAG 检索

## 八、关键查询示例（供开发参照）

### 8.1 查某用户今日待办任务

```sql
SELECT t.id, t.title, t.description, t.verification_criteria, p.name AS project_name
FROM tasks t
JOIN projects p ON p.id = t.project_id
WHERE t.user_id = :uid              -- 通过 project 间接关联，或冗余 user_id
  AND t.due_date = CURRENT_DATE
  AND t.status IN ('pending','failed')
  AND t.deleted_at IS NULL
ORDER BY t.sort_order;
```

### 8.2 知识库全文检索（带高亮）

```sql
SELECT id, title,
       ts_headline('simple', content, q) AS highlight,
       ts_rank(search_tsv, q) AS rank
FROM knowledge_entries, plainto_tsquery('simple', unaccent(:kw)) q
WHERE user_id = :uid
  AND deleted_at IS NULL
  AND search_tsv @@ q
ORDER BY rank DESC
LIMIT 20 OFFSET :offset;
```

### 8.3 语义检索（向量相似度）

```sql
SELECT id, title,
       1 - (vector_embedding <=> :query_vec) AS similarity
FROM knowledge_entries
WHERE user_id = :uid AND deleted_at IS NULL AND vector_embedding IS NOT NULL
ORDER BY vector_embedding <=> :query_vec
LIMIT 10;
```

### 8.4 混合检索（BM25 + 向量 RRF）

```sql
-- RRF: score = Σ 1/(k + rank_i)，k=60
WITH bm25 AS (
  SELECT id, row_number() OVER (ORDER BY ts_rank(search_tsv, q) DESC) AS r
  FROM knowledge_entries, plainto_tsquery('simple', unaccent(:kw)) q
  WHERE user_id = :uid AND search_tsv @@ q LIMIT 100
),
vec AS (
  SELECT id, row_number() OVER (ORDER BY vector_embedding <=> :qv) AS r
  FROM knowledge_entries
  WHERE user_id = :uid AND vector_embedding IS NOT NULL
  ORDER BY vector_embedding <=> :qv LIMIT 100
)
SELECT ke.id, ke.title,
       COALESCE(1.0/(60+b.r),0) + COALESCE(1.0/(60+v.r),0) AS rrf_score
FROM knowledge_entries ke
LEFT JOIN bm25 b ON b.id = ke.id
LEFT JOIN vec  v ON v.id = ke.id
WHERE b.id IS NOT NULL OR v.id IS NOT NULL
ORDER BY rrf_score DESC LIMIT 20;
```

### 8.5 连续打卡更新（事务内）

```sql
INSERT INTO user_scores (user_id, project_id, total_points, streak_days, last_completed_date)
VALUES (:uid, :pid, :score, 1, CURRENT_DATE)
ON CONFLICT (user_id, project_id) DO UPDATE
  SET total_points   = user_scores.total_points + EXCLUDED.total_points,
      streak_days    = CASE
        WHEN user_scores.last_completed_date = CURRENT_DATE THEN user_scores.streak_days
        WHEN user_scores.last_completed_date = CURRENT_DATE - INTERVAL '1 day' THEN user_scores.streak_days + 1
        ELSE 1
      END,
      last_completed_date = CURRENT_DATE;
```

## 十、安全与运维要点

1. **密钥字段**：`user_ai_config` 的 `*_encrypted` 字段禁止出现在日志、异常信息、API 响应中；查询用专用 DTO。
2. **行级隔离**：所有业务表查询必须带 `user_id = current_user_id`，可在 JPA 层用 Hibernate Filter 统一注入。
3. **向量索引**：HNSW 参数 `m=16, ef_construction=64`，查询时 `SET hnsw.ef_search = 100`。
4. **全文检索**：用 `simple` 配置 + `unaccent`，避免中文分词器依赖；中文分词后续可换 `zhparser` 扩展。
5. **备份**：`pg_dump` 每日全量 + WAL 归档；MinIO 用 mc mirror 异地同步。
6. **迁移**：Flyway `baselineOnMigrate=true`，生产环境禁用 `clean`。
7. **监控**：`pg_stat_statements` 开启，慢查询 > 200ms 告警。
